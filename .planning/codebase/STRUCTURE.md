## File Roles in Detail

### `server.js`

The single backend file. Contains:
1. `pg.Pool` instantiation with hardcoded credentials
2. `express()` app creation
3. `app.use(express.static(__dirname))` — serves `index.html`
4. `app.use(express.json())` — parses JSON request bodies
5. `app.get('/tasks', ...)` — SELECT all tasks
6. `app.post('/tasks', ...)` — INSERT new task
7. `app.put('/tasks/:id', ...)` — UPDATE task status
8. `app.delete('/tasks/:id', ...)` — DELETE task
9. `app.listen(port)` — start server

Estimated size: ~60-80 lines

### `index.html`

The single frontend file. Contains:
1. `<head>` with inline `<style>` — basic 3-column CSS layout
2. `<body>` with:
   - Input field + "Add Task" button at top
   - Three-column container: "To Do", "In Progress", "Done"
3. Inline `<script>` with:
   - `fetchTasks()` — GET /tasks, filter by status, render cards
   - `addTask()` — POST /tasks, then fetchTasks()
   - `updateStatus(id, status)` — PUT /tasks/:id, then fetchTasks()
   - `deleteTask(id)` — DELETE /tasks/:id, then fetchTasks()
   - `DOMContentLoaded` listener to call `fetchTasks()`

Estimated size: ~80-100 lines

### `package.json`

Minimal manifest:
- `name`, `version`, `description`
- `dependencies`: `express` and `pg`
- Optional: `scripts.start` pointing to `node server.js`

Estimated size: ~10-15 lines

## Where to Add New Code

**New endpoint:**
- Add route handler directly in `server.js` alongside existing 4 routes.
- Do NOT create a routes directory or separate module.

**New frontend feature:**
- Add HTML elements in `index.html` body.
- Add CSS rules in the inline `<style>` block.
- Add JS functions in the inline `<script>` block.
- Do NOT create separate CSS or JS files.

**New database column:**
- Alter the `tasks` table directly via `psql`.
- Update relevant SQL queries in `server.js` route handlers.
- Update frontend rendering in `index.html` script block.

**New npm dependency:**
- Add to `package.json` dependencies.
- Run `npm install`.
- Import at the top of `server.js` (or in `index.html` script if client-side).

**Guidance:** Do not introduce new files or directories for application code. The spec mandates a 3-file structure. Any new logic belongs in `server.js` (backend) or `index.html` (frontend).

## What Should NOT Exist

The following should **never** be created in this project per spec:
- `src/` directory or any subdirectory for source code
- `public/` or `static/` directory (express.static serves from root)
- `routes/`, `controllers/`, `services/`, `models/`, `middleware/` directories
- Separate `.css` or `.js` files
- `.env` files (credentials are hardcoded)
- `Dockerfile` or `docker-compose.yml`
- Test files (`*.test.js`, `*.spec.js`, `__tests__/`)
- CI/CD config (`.github/`, `.gitlab-ci.yml`)
- `tsconfig.json` or any TypeScript config (plain JS only)
- `.eslintrc`, `.prettierrc`, or other linting config
- `README.md` or documentation beyond `Docs/Spec.md`

## Setup Commands

Per spec, three terminal commands to get running:

```bash
# 1. Create the PostgreSQL database and table
createdb kanban_practice
psql -d kanban_practice -c "CREATE TABLE tasks (id SERIAL PRIMARY KEY, title VARCHAR(255) NOT NULL, status VARCHAR(20) DEFAULT 'todo', created_at TIMESTAMP DEFAULT NOW());"

# 2. Install dependencies
npm install

# 3. Start the app
node server.js
```
